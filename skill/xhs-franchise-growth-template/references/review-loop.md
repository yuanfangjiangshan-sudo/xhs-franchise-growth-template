# Review Loop

Use this when the user asks for weekly review, monthly review, or rule updates.

## Inputs

Ask for:

- recent note list
- topic/title
- impressions or views
- likes
- saves
- comments
- follows
- private-message or keyword data, if available

## Review Structure

1. Overall account movement
2. Best-performing content
3. Weakest content
4. Repeated audience pains
5. Title and cover patterns
6. Conversion signals
7. Next-week topic priorities
8. Proposed skill-rule updates

## Rule Update Principles

Only update durable rules:

- repeatable topic pattern
- title pattern
- risk boundary
- source-selection rule
- output format improvement

Do not overfit to one note unless the signal is very strong.
